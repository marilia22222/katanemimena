# katanemimena
ΠΕΡΙΓΡΑΦΗ ΠΡΟΒΛΗΜΑΤΟΣ
Στο συγκεκριμένο πρόβλημα κληθήκαμε να αναπτύξουμε ένα μοντέλο πελάτη
- εξυπηρετητή - εξυπηρετητή και αφορά μια απλοποιημένη εφαρμογή η οποία
θα δίνει τη δυνατότητα αγοράς εισιτηρίων για διάφορα θεάματα όπως θεατρικές
παραστάσεις, συναυλίες, αθλητικά γεγονότα και άλλες εκδηλώσεις μέσω του
διαδικτύου.
Προκειμένου να το επιτεύξουμε αυτό αποφασίσαμε να χρησιμοποιήσουμε 4
java project.
Χρειαζόμαστε ένα το οποίο ονομάσαμε server 1, που θα λειτουργήσει σαν τον
πρώτο μας εξυπηρετητή ο οποίος διαχειρίζεται το κεντρικό σύστημα κρατήσεων.
Το δεύτερο το ονομάσαμε server 2 το οποίο είναι ο εξυπηρετητής 2 ο οποίος
διαχειρίζεται την βάση δεδομένων.
Το τρίτο είναι ο χρήστης TicketClient ο οποίος συνδέεται στον server 1.
Το τέταρτο είναι μια βιβλιοθήκη την οποία ονομάσαμε cinema που έχει όλα τα
χαρακτηρίστηκα που κανονικά θα εμπεριέχονταν σε όλα τα project αλλά για
ευκολία το κάναμε import σαν βιβλιοθήκη.
Το project client ήταν να είναι οι πελάτες αλλά επειδή υπήρχε θέμα με τα import
των JavaFX καταλήξαμε να κάνουμε το περιβάλλον με Swings.
Αρχικά τρέχουμε τον server 2 για να ετοιμάσει την επικοινωνία με τον server 1,
έπειτα τρέχουμε τον server 1, η επικοινωνία γίνεται με sockets. Τώρα μπορούμε
να βάλουμε τους χρήστες, οπότε τρέχουμε το TicketClient.
Εδώ κανονικά έπρεπε να γίνει χρήση RMI, για λογούς που δεν μπορούσαμε να
καταλάβουμε δεν καταφέραμε να το υλοποιήσουμε έτσι, οπότε βάλαμε πάλι
sockets.
Οι βασικές λειτουργίες που περιέχει η εφαρμογή είναι:
Η δυνατότητα εγγραφής αλλά και διαγραφής ενός χρήστη
Αναζήτηση και Ακύρωση Παραγγελίας,
Με αποσύνδεση μας βγάζει στο αρχικό login.
Ως Admin μπορούμε να κανουμε:
Καταχώριση θεάματος, Καταχώριση Παράστασης και Προβολή θεαμάτων.
